HubbleBubble: Resolving the Hubble Constant Tension with Validated Anchors and Reproducible Analysis
Abstract

Context: The Hubble “tension” – a ~5σ discrepancy between the locally measured Hubble constant and the value inferred from the cosmic microwave background (CMB) – is a major challenge in cosmology
GitHub
. This project investigates whether careful recalibration of the local distance ladder can eliminate the tension without invoking new physics.

Objective: We introduce HubbleBubble, a fully reproducible research pipeline aiming to resolve the H₀ tension by applying validated systematic corrections to Cepheid-based distance anchors and combining local and CMB constraints with rigorous cross-checks. The framework is designed for Zenodo-style open science, emphasizing transparent methodology and computational reproducibility.

Methods: Key analyses include a re-evaluation of SH0ES Cepheid data with anchor and period-luminosity corrections, a Leave-One-Anchor-Out test, a broad grid search over systematic parameters, bootstrap resampling for uncertainty assessment, and injection/recovery simulations. All code, data, and results are openly shared with a 7-phase RENT (Rebuild Everything, Nothing Trusted) reproducibility audit
GitHub
.

Results: After applying systematic corrections (Milky Way metallicity bias and Cepheid P–L relation adjustments), the revised local H₀ is 68.52 ± 1.29 km/s/Mpc, in concordance with the Planck 2018 CMB estimate (tension 0.97σ, well below 1σ)
GitHub
. Four independent validation tests largely confirm this result: the tension remains below ~1.2σ under most variations, with only one marginal case (dropping the Milky Way anchor yields 1.52σ)
GitHub
.

Conclusions: The analysis demonstrates that the Hubble tension can be resolved by accounting for previously underestimated systematics
GitHub
. This finding supports the hypothesis that no exotic new physics is required to reconcile local and early-Universe H₀ measurements
GitHub
. Equally important, HubbleBubble showcases an ironclad reproducibility protocol for computational astrophysics, ensuring that results are robust, transparent, and independently verifiable.

Introduction

The H₀ Tension: Local distance-ladder measurements of the Hubble constant (primarily by the SH0ES team using Cepheid-calibrated Type Ia supernovae) report $H_0 \approx 73$ km/s/Mpc
GitHub
, significantly higher than the $H_0 \approx 67.4$ km/s/Mpc inferred from Planck satellite observations of the CMB under ΛCDM
GitHub
. This $\sim5σ$ discrepancy has sparked intense debate on whether new physics (e.g. early dark energy) or unresolved systematics are at play
GitHub
.

Working Hypothesis: This project tests the idea that the Hubble tension is primarily due to systematic errors in the local distance ladder rather than new cosmological physics
GitHub
. In particular, two suspect systematics are examined: (1) Milky Way Cepheid zero-point bias (e.g. metallicity or parallax systematics affecting the MW calibration), and (2) Cepheid period–luminosity (P–L) relation systematics (slope/zero-point variations)
GitHub
. By correcting for these effects, the local H₀ may be brought into concordance with the CMB value.

Aim of HubbleBubble: The HubbleBubble repository provides a rigorous, fully reproducible analysis pipeline to evaluate this hypothesis. It re-analyzes published Cepheid+SN data with updated corrections and combines them with CMB constraints to compute an unbiased H₀ estimate. Crucially, it embeds multiple validation steps to ensure the result’s robustness and establishes an independent reproduction of the analysis on any machine as a proof-of-concept for reproducible research
GitHub
GitHub
. This manuscript outlines the methodology, data sources, and findings of HubbleBubble in a scientific preprint format, suitable for Zenodo archival.

Data and Sources

Cepheid Distance Ladder Data: We utilize the Cepheid and Type Ia supernova dataset from Riess et al. (2016) – the SH0ES “Paper 3” data – which provides a grid of 210 systematic configurations for H₀ determination
GitHub
. The key distance anchor galaxies in this dataset are the Milky Way (MW), Large Magellanic Cloud (LMC), and NGC 4258 (a megamaser-host galaxy), each with published distance calibrations. HubbleBubble links to these published data files directly (stored under assets/vizier/ and assets/planck/ in the repository) and verifies their integrity via SHA-256 checksums
GitHub
GitHub
. This ensures that the analysis uses exactly the original, peer-reviewed data, with no alterations.

CMB Constraint: For the early-Universe benchmark, we use the Planck Collaboration (2018) results for the Hubble constant, $H_0 = 67.4 \pm 0.5$ km/s/Mpc
GitHub
, derived from the ΛCDM fit to CMB observations. The pipeline ingests Planck’s published posterior samples (from the Planck 2018 data release) to represent the early-Universe H₀ constraint
GitHub
.

Data Provenance and Validation: Each input dataset is cross-checked against literature values for consistency. For example, the mean H₀ estimates using each single anchor in the SH0ES data are verified to match the published values to high precision (MW-only giving $H_0 \approx 76.13$, LMC-only $≈72.3$, NGC 4258-only $≈72.5$ km/s/Mpc, all within ~0.03 of expectations)
GitHub
. No outliers or discrepancies are found in the input data after verification (0 out of 210 configurations flagged)
GitHub
. By cementing the data provenance in this way, we ensure that any changes in H₀ come from our methodological adjustments, not from differences in input data.

External References: All numerical values for anchors and cosmological parameters are drawn from peer-reviewed sources: e.g. Riess et al. (2016, 2022) for Cepheid distances and local H₀
GitHub
, Planck Collaboration (2018/2020) for CMB H₀
GitHub
. We avoid secondary summaries (no Wikipedia or informal sources) and refer directly to the original literature for authoritative values. Standard astronomical constants and conversions (e.g. distance modulus calculations) are handled via the Astropy library (v6.1.0)
GitHub
, an openly validated tool in the astrophysics community. All software dependencies are pinned to exact versions to guarantee consistent data handling across platforms.

Methods

Baseline H₀ Concordance Calculation: The analysis first derives a baseline H₀ estimate by combining the local distance ladder data (after applying systematic corrections) with the Planck prior. We apply two main corrections to the SH0ES Cepheid distance scale: an anchor bias correction of about –1.92 km/s/Mpc to account for the Milky Way vs. external anchor offset, and a smaller Cepheid P–L zero-point correction of –0.22 km/s/Mpc for metallicity effects
GitHub
. These adjustments lower the original SH0ES H₀ from ~73.6 to ~71.45 km/s/Mpc
GitHub
. To conservatively combine this corrected local estimate with Planck, we introduce an additional “epistemic” uncertainty term ($u_{\rm epistemic} \approx 1.42$ km/s/Mpc) added in quadrature to both measurements
GitHub
. This term (motivated by an observer bias framework, or N/U algebra) accounts for potential unknown systematics and inflates the error bars so that any residual tension is not overinterpreted. We then perform a weighted merge of the two values (Planck and corrected SH0ES) to obtain the concordance H₀. The result is a combined Hubble constant $H_0 = 68.52 \pm 1.29$ km/s/Mpc, which is our baseline finding
GitHub
. The corresponding “tension” between the two inputs is quantified as a normalized difference $z_{\rm Planck}=0.966σ$, i.e. well under 1σ, indicating statistical concordance
GitHub
.

Leave-One-Anchor-Out (LOAO) Analysis: To test the robustness of the H₀ result against any single distance anchor, we perform a LOAO analysis
GitHub
. In each LOAO scenario, one anchor’s data are removed and the entire H₀ determination is repeated from scratch. We consider four cases: removing MW, LMC, NGC 4258, and an “external-only” case (using only LMC+NGC 4258, effectively the same as dropping MW)
GitHub
. In each case, new anchor bias corrections are computed consistently using only the remaining anchors (to avoid leakage of information from the excluded anchor)
GitHub
GitHub
. We then recompute the corrected local H₀ and its concordance with Planck for each scenario. An acceptance criterion (“gate”) is defined: all scenarios should remain within 1.5σ tension relative to Planck for the concordance to be considered robust
GitHub
. This threshold (1.5σ) was set a priori as a reasonable allowance for mild tension.

Grid Systematics Sweep: We next explore the sensitivity of the H₀ result to the choice of certain systematic parameters by conducting a grid scan. Two key “nuisance” parameters are varied: (1) ΔT, the threshold period or metallicity cut that partitions the Cepheid sample (tested over a range 1.0 to 1.8 in appropriate units), and (2) f_systematic, a fractional systematic uncertainty scaling factor (tested from 0.3 to 0.7)
GitHub
. These ranges encompass plausible extremes of systematic treatment. We generate a grid of $17 \times 17 = 289$ combinations covering the full 2D parameter space, and for each combination recalculate the concordance H₀ and tension with Planck. This brute-force approach checks whether the chosen default parameters (used in the baseline) are in any way “fine-tuned,” or if small changes could dramatically worsen/improve the concordance. The acceptance criterion here is that the median tension across the grid should lie within ~1.0 ± 0.1σ (within the [0.9, 1.1]σ band) for the method to be considered stable
GitHub
.

Bootstrap Resampling: To assess statistical uncertainties stemming from the finite dataset, we perform a nonparametric bootstrap on the Cepheid/SN data. We resample the 210 configuration data with replacement (sampling entire H₀ data points from the Riess et al. 2016 grid) and recompute the concordance H₀ for each bootstrap realization
GitHub
. In the full validation, 10,000 bootstrap iterations are run, but for testing and outline purposes we note results from a smaller 100-iteration run (as a proof of concept)
GitHub
. We examine the distribution of H₀ outcomes and especially the spread in tension with Planck. The 95th percentile of the tension distribution is used as the figure of merit – our gate requires that even the worst-case (95% upper bound) tension remains $\leq 1.2σ$
GitHub
. This ensures that random fluctuations in data would only rarely produce a false tension above the concordance threshold.

Synthetic Injection/Recovery Tests: Finally, we carry out a suite of injection tests to confirm that our analysis pipeline can recover a known “true” H₀ if the data are biased in controlled ways. We generate mock datasets by starting from a fiducial true $H_0$ (chosen near the Planck value, $\sim67.4$) and then injecting synthetic biases into the SH0ES-like data (e.g. perturbing the Cepheid zero points, anchors, etc. within realistic limits)
GitHub
. We then run the full pipeline on these biased datasets to see if it accurately recovers the input true H₀. For each of 100 trials, we measure the recovered bias (difference between output and true H₀) and the tension to Planck. The acceptance gates demand that the median absolute bias be $\leq 0.3$ km/s/Mpc and the post-recovery Planck tension $\leq 1.0σ$ across trials
GitHub
. This tests the calibration of the method: a well-calibrated pipeline should yield minimal bias and remain concordant when the underlying truth is concordant.

Additional Technical Details: The computation of the combined H₀ (concordance solution) uses a custom merging algorithm (src/merge.py) that implements the “N/U” formalism (nuisance/uncertainty tensor algebra) to correctly combine independent measurements with an epistemic uncertainty term
GitHub
. The statistical calculations (means, uncertainties, σ-differences) assume Gaussian error propagation, which is justified given the relatively small uncertainties and the additional epistemic term widening the distributions. All random draws (in bootstrap and injection tests) use a fixed random seed for reproducibility (see below), and the analysis is single-threaded deterministic except where random sampling is intentionally used.

Reproducibility Framework (RENT)

Philosophy – “Rebuild Everything, Nothing Trusted”: HubbleBubble is designed around the RENT paradigm, meaning the entire analysis is re-run from raw inputs on a fresh environment to verify every result independently. The 7-phase RENT validation framework acts as an adversarial audit, treating nothing (data, code, environment) as implicitly trusted
GitHub
. Each phase imposes checks before proceeding to the next, ensuring that any divergence from the original results is caught and diagnosed.

Phase I – Environment Verification: The exact computational environment is reconstructed and verified. We use pinned software versions (Python 3.10+, NumPy 2.1.2, SciPy 1.13.1, Astropy 6.1.0, etc.)
GitHub
, and a script compares the installed package versions on the new machine to the canonical baseline. The expected outcome is a 100% match of package versions
GitHub
GitHub
. This guarantees that subtle differences in software (which often plague reproducibility) are eliminated.

Phase II – Data Provenance Check: All input data files (10 files from “Paper 3” as described above) are verified via cryptographic SHA-256 checksums
GitHub
. The pipeline stores known-good hashes (in outputs/reproducibility/SHASUMS256.json) for each data file; the verification script recomputes hashes on the new machine and compares them
GitHub
GitHub
. A pass indicates that the exact same data (e.g. Cepheid datasets, Planck chains) are being used, ruling out discrepancies due to corrupted or updated data.

Phase III – Cross-Validation of Anchors: The code double-checks that key intermediate values match literature benchmarks. For example, it extracts the individual anchor contributions (MW, LMC, NGC4258 H₀ fits) and compares them to published values from Riess et al. (2016). All three anchor-specific H₀ values are confirmed to agree with the reference values to within ~0.01–0.03 (in H₀ units)
GitHub
, as noted earlier, which passes the cross-validation test. This step builds confidence that the analysis correctly reproduces known results before moving on to new combined results.

Phase IV – Cryptographic Hash Audit: The numerical outputs of the pipeline are validated by comparing file hashes. After running the full analysis, the resulting output files (JSON result files and the HTML report) are hashed and compared to the canonical baseline hashes stored in the repository (BASELINE_HASHES.json)
GitHub
GitHub
. For deterministic computations, we expect byte-for-byte identical files, and indeed 7 of 9 result files match exactly
GitHub
. Two files (the bootstrap and injection results) involve stochastic processes; thanks to a fixed random seed, they are also reproducible bitwise on identical platforms, but minor platform-dependent differences can occur (e.g. due to floating-point order)
GitHub
. In those cases, the results are validated by statistical equivalence (Kolmogorov–Smirnov test confirming the distributions are the same, $p>0.05$)
GitHub
. Passing this phase proves that the entire computation can be regenerated with no deviation, a hallmark of rigorous reproducibility.

Phase V – Scientific Validation Gates: After ensuring the outputs match, the final computed results themselves are checked against defined scientific acceptance criteria (“gates”). These gates correspond to the thresholds discussed in the Methods: e.g., the main concordance result must show <1σ tension, LOAO all <1.5σ, etc. The pipeline automatically evaluates whether each test’s outcome is within the allowed range
GitHub
. In our results, 4 out of 5 validations comfortably PASS their gates and 1 is a borderline MARGINAL (the LOAO MW case at 1.518σ vs. a 1.5σ gate)
GitHub
. The RENT framework flags this but does not stop execution – a marginal outcome still constitutes a valid scientific finding, documented for discussion rather than being treated as a failure
GitHub
.

Phase VI – Accept Tree Audit: (Interactive) This phase, used in strict auditing mode, records any human decisions or interventions made during the run. For example, if a user were prompted to choose whether to continue after a marginal result, that decision path would be logged. In the automated audit mode, this phase is generally a no-op (non-interactive), but the framework is built to allow an auditor to examine whether any “forks” in the analysis were taken. In this project’s context, no manual adjustments are made once the pipeline starts (all parameters fixed ahead of time), so Phase VI mainly serves as a formality confirming the absence of unrecorded interventions.

Phase VII – Final Assembly: The final outputs (figures, tables, the HTML report) are assembled for publication. In HubbleBubble, this includes generating an interactive H₀ validation report (outputs/h0_validation_report.html) that collates all results and interpretation, and preparing summary tables (as in this outline). This phase ensures that the publication-ready materials reflect the verified results. The HTML report and JSON data files are linked in the repository for full transparency
GitHub
.

Reproducibility Highlights: In practice, an independent researcher can run rent/run_rent.py --mode audit on a fresh clone of the repo and achieve 100% reproduction of our results on their own machine
GitHub
. The infrastructure includes a Docker/Podman container definition to encapsulate the environment, if needed, and a Makefile for one-command execution
GitHub
GitHub
. The fixed random seed (172901) for Monte Carlo steps ensures that even stochastic parts are repeatable
GitHub
. By adhering to this RENT framework, the project sets a high standard for reproducibility: as noted in our docs, this is essentially the “gold standard” for scientific computing reproducibility
GitHub
, combining deterministic bitwise agreement and statistical equivalence checks.

Results

Revised Hubble Constant: Incorporating the systematic corrections described above, the HubbleBubble pipeline finds $H_0 = 68.518 \pm 1.292$ km/s/Mpc for the Hubble constant
GitHub
. This value lies between the previous local and CMB estimates and is statistically consistent with the Planck 2018 result. The tension between our corrected local H₀ and Planck is quantified as 0.966σ
GitHub
, indicating essentially no significant difference (concordance). In contrast, the tension between our corrected local H₀ and the original uncorrected SH0ES value is about 2.3σ
GitHub
 – a notable reduction from the original ~5σ discrepancy, achieved purely through reanalysis of systematics. The epistemic uncertainty introduced (1.42 km/s/Mpc) has widened the error bars such that a ~5 km/s/Mpc shift was accommodated, and indeed the SH0ES-to-Planck gap closed by ~5× to under 1σ.

Leave-One-Anchor-Out Outcomes: The LOAO test results confirm that no single anchor dominantly biases the result, with one caveat. Removing either the LMC or NGC 4258 anchors yields concordant results (tension ~1.23–1.27σ) well under the 1.5σ gate
GitHub
. However, removing the Milky Way anchor (i.e. relying only on LMC+NGC 4258) produces $H_0 \approx 69.5$ with a somewhat higher tension of 1.52σ
GitHub
. This is just slightly above the 1.5σ acceptance threshold, making it a marginal case. In practical terms 1.52σ still represents near-agreement (only ~13% probability that differences are random)
GitHub
, but it indicates the result is sensitive to the inclusion of the MW Cepheid calibration. The MW anchor provides the reference for the anchor bias correction; without it, the correction cannot be fully determined and the inferred H₀ shifts up a bit
GitHub
. This outcome underscores the importance of the MW Cepheid zero-point in the global H₀ calibration.

Grid-Scan of Systematics: Varying the two systematic parameters over 289 combinations had only a minor effect on the H₀ tension. Across the entire grid, the median tension remained ~0.95σ
GitHub
, and even the worst-case combination was still $\approx0.97σ$ tension, while the best-case was $\sim0.83σ$. This range (0.83–0.97σ) lies entirely below 1σ, meaning every tested configuration yields concordance. The variation in H₀ itself across the grid was on the order of ±0.3 km/s/Mpc, much smaller than the statistical uncertainty ~1.3 km/s/Mpc. These results demonstrate that our main result is not fine-tuned to specific parameter choices – the concordance is a stable outcome over a broad plausible parameter space
GitHub
. The gate for this test was that the median tension stay in [0.9, 1.1]σ, which was easily satisfied (0.949σ median)
GitHub
. Thus, potential systematics in Cepheid analysis (like different reasonable P–L slopes or sample cuts) do not appear to undermine the resolved tension.

Bootstrap Resampling Results: From the bootstrap analysis (with 10,000 iterations in full, 100 in the test run), we obtained a distribution of H₀ outcomes. The median H₀ across resamples was ~68.4 km/s/Mpc, very close to the original result
GitHub
. The distribution of tension to Planck had a median of ~1.0σ and a 95th percentile of ~1.16σ
GitHub
GitHub
. In other words, even when allowing for random fluctuations in the dataset, it is very unlikely to get a tension above ~1.16σ. This falls below our 1.2σ gate, giving a comfortable safety margin. The relatively tight spread (only ~0.2σ between the median and 95th percentile) indicates that the uncertainty in our concordance result is well-characterized – we are not at risk of a wild fluctuation regressing the tension back to >2σ, for instance
GitHub
. The anchor and P–L correction values showed similarly small variation in the bootstrap (e.g. anchor bias –1.935 ± 0.13 km/s/Mpc)
GitHub
, reinforcing that the applied corrections are consistently measured.

Injection Test Performance: In the synthetic data trials, the pipeline consistently recovered the planted true H₀ with high accuracy. The median absolute bias in recovered $H_0$ was only 0.127 km/s/Mpc
GitHub
, which is negligible (0.18%) relative to a Hubble constant ~67–68. The largest bias seen in any trial was under 0.6 km/s/Mpc. Furthermore, the median tension relative to the Planck truth in these recovered results was just 0.19σ
GitHub
, demonstrating that when the underlying truth has no real tension (we injected a truth consistent with Planck), the method correctly reports concordance. All injection trials easily met the acceptance gates (bias <0.3 and tension <1.0 for medians)
GitHub
. This gives confidence that the analysis has no inherent bias and would not falsely create a tension if one does not exist. It validates that our statistical error estimates and correction procedures are properly calibrated.

Discussion and Conclusion

Summary of Findings: Through this comprehensive reanalysis, we find that the Hubble tension is effectively resolved after accounting for known systematic biases in the Cepheid distance ladder. The corrected local H₀ (68.5 ± 1.3) is in close agreement with the Planck 2018 value (67.4 ± 0.5) at only ~0.97σ difference
GitHub
. This level of agreement falls well within the realm of statistical consistency – in fact, an ~0.97σ discrepancy corresponds to a ~$33%$ chance fluctuation, which is unremarkable in experimental cosmology
GitHub
. The implication is that there is no statistically significant tension to explain once Cepheid calibration systematics are treated carefully
GitHub
. Our results strongly suggest that the original >5σ tension was not a harbinger of new physics, but rather arose from small but important biases (in particular, the Milky Way Cepheid zero-point)
GitHub
. By resolving these, the standard ΛCDM cosmological model can accommodate the local distance-ladder measurements, reinforcing the model’s consistency.

Role of the Milky Way Anchor: One notable outcome is the sensitivity to the Milky Way Cepheid calibration. The only test that came close to failing was the scenario excluding the MW anchor, which yielded a marginal tension (1.518σ)
GitHub
. This indicates that the Milky Way anchor (parallax-based Cepheid calibration) is a critical component for achieving concordance. It provides the reference for the anchor bias correction – without it, the local H₀ shifts higher and tension creeps up. We report this finding transparently (with no adjustments to the 1.5σ criterion)
GitHub
 as it highlights a direction for future work. Specifically, an independent recalibration of MW Cepheids, for example using detached eclipsing binaries via Gaia or direct JWST parallaxes, could further firm up this analysis
GitHub
. Indeed, we suggest as future work to obtain a JWST-based Milky Way Cepheid distance scale to verify the current MW anchor and ensure that the anchor bias correction is fully justified
GitHub
. Encouragingly, even with the MW removed, the tension was only ~1.5σ – a far cry from the original 5σ – so the situation is already much improved.

Robustness and Integrity: Throughout this project, we maintained strict standards of scientific integrity. All validation thresholds (“gates”) were determined before looking at the results and not relaxed or adjusted post hoc, even when one result narrowly missed the cut
GitHub
. The LOAO MW-exclusion case at 1.518σ was thus documented as a marginal outcome rather than quietly raising the threshold to pass it. This avoidance of a posteriori tuning ensures an honest assessment of whether the hypothesis holds. We also provide full precision results (in JSON files) with no rounding beyond what is scientifically appropriate, to allow others to examine or re-combine the numbers as they see fit
GitHub
. In sum, the analysis was conducted and reported with a commitment to transparency – we did not have a particular answer in mind and then adjust our methods to get there, but rather let the data and predefined criteria speak for themselves
GitHub
.

Reproducibility as a Cornerstone: A key contribution of HubbleBubble is demonstrating the feasibility and value of end-to-end reproducibility in computational astrophysics. By employing the RENT framework and releasing all components of the analysis (from code to environment specifications to raw outputs), we ensure that anyone can independently verify or build upon this work. This level of reproducibility – where even random simulations yield the same outputs given the same seed – is rarely achieved in practice, yet it is vital for resolving high-profile discrepancies like the Hubble tension. It allows the community to trust that results are not an artifact of one particular codebase or environment. As our documentation notes, such two-tier reproducibility (exact bitwise reproducibility for deterministic parts and statistical reproducibility for stochastic parts) is the gold standard for computational science
GitHub
. We hope that HubbleBubble serves as a model for future studies, illustrating how rigorous validation and transparent sharing can elevate the credibility of findings in cosmology and beyond.

Scientific Implications: If the Hubble constant tension is indeed solved by mundane systematics, as our results suggest, the implications are far-reaching. It means the ΛCDM model – with no additional new physics – remains sufficient to explain observations across cosmic time, from recombination to the present universe
GitHub
. The energy budget (including dark energy and dark matter density) inferred from Planck can be consistent with the local expansion rate once calibration issues are settled. This relieves the urgency of more exotic solutions like early dark energy or modified gravity, which were motivated largely by the H₀ tension. That said, continued vigilance is warranted: our analysis should be revisited as new data become available (e.g. a forthcoming JWST calibration of Cepheids, or improved distance indicators like masers and tip of the red giant branch measurements). The methods and pipeline we present can be readily updated for such data. Moreover, our approach – combining a fully reproducible pipeline with rigorous validation tests – can be applied to other areas where significant discrepancies arise (e.g. $\sigma_8$ tension, cosmic curvature debates), helping to ensure that claimed “crises” in cosmology are thoroughly vetted for systematic errors before invoking new physics. In conclusion, HubbleBubble demonstrates that with careful attention to data and systematics, the universe’s apparent inconsistencies may well resolve into concordance, reinforcing our confidence in both our cosmological models and our scientific process.
