  Missing Quadratic Formula Explanation

  The Problem: The original U/N multiplication was missing the quadratic uncertainty terms (u × u terms).

  What Was Missing:
  # BEFORE (incomplete):
  u_t_result = |n_a1|u_t2 + |n_a2|u_t1 + cross_terms

  # AFTER (complete with quadratic terms):
  u_t_result = |n_a1|u_t2 + |n_a2|u_t1 + cross_terms + quad_u_t + quad_mixed
  u_m_result = |n_m1|u_m2 + |n_m2|u_m1 + quad_u_m

  The Quadratic Terms Added:
  1. quad_u_t = u_t1 × u_t2 (tolerance-tolerance interaction)
  2. quad_u_m = u_m1 × u_m2 (measurement-measurement interaction)
  3. quad_mixed = u_t1 × u_m2 + u_m1 × u_t2 (cross-tier interaction)

  Why It Matters: These terms ensure conservativity - when multiplying uncertain quantities, the uncertainties compound quadratically. Without them, we underestimate the result's uncertainty, violating
  Theorem 9.

class UNReanchor:
    def __init__(self, datasets):
        """datasets: list of (name, n, u) tuples"""
        self.datasets = datasets

    def reanchor(self, weights=None, shared=None):
        # 1. compute shared UNA
        if shared is None:
            if weights is None:
                n_anchor = sum(n for _, n, _ in self.datasets) / len(self.datasets)
            else:
                w_sum = sum(weights)
                n_anchor = sum(w*n for (_, n, _), w in zip(self.datasets, weights)) / w_sum
            u_t = max(u for _, _, u in self.datasets)
        else:
            n_anchor, u_t = shared

        # 2. adjust uncertainties
        adjusted = []
        for name, n, u in self.datasets:
            d = abs(n - n_anchor)
            u_prime = u + max(0, d - (u_t + u))
            adjusted.append((name, n, u_prime))
        return n_anchor, u_t, adjusted
